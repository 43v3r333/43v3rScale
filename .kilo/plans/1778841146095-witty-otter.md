# Plan: Clone and Run Understand-Anything

Repo: https://github.com/Lum1104/Understand-Anything

## What It Is

A Claude Code Plugin (also supports Codex, Cursor, Copilot, Gemini CLI, OpenCode, etc.) that analyzes any codebase with a multi-agent pipeline, builds an interactive knowledge graph, and serves a dashboard. Monorepo managed with pnpm workspaces.

## Prerequisites

- **Node.js** (v20+ recommended — project uses `"engines"` field)
- **pnpm** 10.x (`corepack enable` or `npm i -g pnpm`)
- **Git**
- **TypeScript** 5.7+ (comes as devDependency)
- **Workaround needed**: On newer Node.js with built-in corepack, pnpm's signature verification may fail with `Cannot find matching keyid`. Fix: `pnpm config set verify-signatures false` or set env `PNPM_COREPACK_SKIP_SIGNATURE_CHECK=1` before any pnpm command. If that doesn't help, use the `--no-verify-signatures` flag or disable corepack entirely: `corepack disable pnpm` and install pnpm via npm (`npm i -g pnpm@10`).

## Steps

### 1. Clone

```bash
git clone https://github.com/Lum1104/Understand-Anything.git
cd Understand-Anything
```

### 2. Install dependencies

```bash
pnpm install
```

This installs all workspace packages. The workspace layout:

- `understand-anything-plugin/` — the core plugin package (`@understand-anything/skill`)
- `homepage/` — Astro-based documentation site

### 3. Build

```bash
pnpm build
```

This runs `pnpm --filter @understand-anything/core build` first (compiles the core library), then builds the plugin (`tsc`) and homepage. Alternatively:

```bash
# Build just the plugin
pnpm --filter @understand-anything/skill build
```

### 4. Run tests

```bash
pnpm test
```

Uses Vitest v3. Runs all workspace tests.

### 5. Run the dashboard (dev)

```bash
pnpm dev:dashboard
```

This starts the Astro homepage dev server (not the plugin itself — the plugin is loaded inside a Claude Code / IDE session).

### 6. Install the plugin for your IDE

**Claude Code (native):**
```
/plugin marketplace add Lum1104/Understand-Anything
/plugin install understand-anything
```

**VS Code + GitHub Copilot (v1.108+):**
Just open the repo in VS Code — auto-discovers `.copilot-plugin/plugin.json`.

**Manual one-line install (Codex, OpenCode, etc.):**
```bash
curl -fsSL https://raw.githubusercontent.com/Lum1104/Understand-Anything/main/install.sh | bash -s codex
# or: opencode, openclaw, antigravity, gemini, pi, vibe, hermes, cline, kimi
```

Installer clones to `~/.understand-anything/repo` and symlinks skills to the platform's skills directory.

### 7. Use it

Once installed and your IDE restarts:

| Command | What it does |
|---------|-------------|
| `/understand` | Scan current project → builds `.understand-anything/knowledge-graph.json` |
| `/understand --language zh` | Same but Chinese output |
| `/understand-dashboard` | Open interactive graph dashboard |
| `/understand-chat` | Ask questions about the codebase |
| `/understand-diff` | Analyze impact of current changes |
| `/understand-explain <file>` | Explain a specific file/function |
| `/understand-onboard` | Generate onboarding guide |
| `/understand-domain` | Extract business domains/flows |
| `/understand-knowledge <path>` | Analyze Karpathy-pattern wiki knowledge base |

## Gotchas

- **Build order matters**: `pnpm build` handles it, but running `pnpm --filter @understand-anything/skill build` alone will fail if `@understand-anything/core` isn't built first. Use workspace-aware commands.
- **`prepare` script**: root `package.json` has `"prepare": "pnpm --filter @understand-anything/core build"` — runs automatically after `pnpm install`.
- **`onlyBuiltDependencies`**: pnpm config in root `package.json` lists tree-sitter parsers and esbuild/sharp as build-only deps that must be pre-built.
- **Graph data lives in `.understand-anything/knowledge-graph.json`** — commit this to share with teammates; exclude `.understand-anything/intermediate/` and `.diff-overlay.json`.
- **Large graphs (10 MB+)**: use git-lfs for the JSON files.
- **No local "run" command**: the plugin is consumed inside an AI IDE session, not as a standalone server. The dashboard is a static Astro site that reads the graph JSON.
- **esbuild is a built dependency**: if you hit errors about missing `esbuild`, run `pnpm install` again to trigger the postinstall script.

## Architecture Summary (for ramp-up)

```
understand-anything-plugin/     # Main plugin package
  packages/
    core/                        # Shared analysis logic, graph types
    skills/                      # Individual skill modules (understand, dashboard, etc.)
    skill/                       # Plugin entry point
homepage/                        # Docs site (Astro)
```

TypeScript 5.7+, strict mode, ES2022 target, ESNext modules, `vitest` for testing.
